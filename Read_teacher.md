# 📋 Розумник — Інструкція для вчителя
## Як створювати тести

---

## Структура файлу тесту (.xlsx)

Кожен тест — це один Excel-файл з **двома аркушами**:

### Аркуш 1 — `Мета`
Містить загальну інформацію про тест.

| A (ключ)  | B (значення)         |
|-----------|----------------------|
| Тема      | Будова клітини       |
| Предмет   | Біологія             |
| Клас      | 6                    |
| Питань    | 15                   |

> **Питань** — скільки питань побачить учень. Рекомендовано: 15–20.

---

### Аркуш 2 — `Питання`
Містить банк питань. Колонки:

| Колонка              | Опис                                                                 |
|----------------------|----------------------------------------------------------------------|
| `ID`                 | Порядковий номер (1, 2, 3...)                                        |
| `Група`              | Номер групи варіантів питання (пояснення нижче)                     |
| `Тип`                | `single` — одна відповідь, `multiple` — кілька, `truefalse` — правда/хибно |
| `Питання`            | Текст питання                                                        |
| `Варіант_А`          | Варіант відповіді А                                                  |
| `Варіант_Б`          | Варіант відповіді Б                                                  |
| `Варіант_В`          | Варіант відповіді В                                                  |
| `Варіант_Г`          | Варіант відповіді Г                                                  |
| `Правильна_відповідь`| Літера правильної відповіді: `А`, `Б`, `В` або `Г`. Для `multiple`: `Б,В` |

---

## 🔑 Головна ідея: Групи питань

**Група** — це кілька питань про одне й те саме, але сформульованих по-різному.

Система автоматично обирає **лише ОДНЕ** питання з кожної групи — випадково.

### Навіщо це потрібно?

Учень, який вирішить пройти тест двічі:
- 1-й раз: побачить «Хто відкрив клітину?» → запише відповідь «Б»
- 2-й раз: побачить «Клітину вперше описав учений на прізвище:» → його підказка «Б» вже не відповідає тій самій позиції

Варіанти відповідей також **перемішуються** при кожному проході.

### Приклад групи:

```
ID  Група  Питання
1   1      Що є основною одиницею живих організмів?       → Б (Клітина)
2   1      З чого складаються всі живі організми?         → В (З клітин)
3   1      Яка структура є найменшою, здатною до життя?   → Г (Клітина)
```

Учень побачить лише ОДНЕ з цих трьох — рандомно.

## ✅ Рекомендації

### Скільки питань готувати?

| Показується учню | Рекомендовано в банку |
|------------------|-----------------------|
| 15 питань        | 30–40 питань          |
| 20 питань        | 40–60 питань          |

### Скільки варіантів у групі?

Оптимально **2–4 варіанти** на групу. Більше — зайва робота.

### Питання без групи

Якщо питання унікальне і немає сенсу його перефразовувати — залиште `Група` **порожньою**. Таке питання завжди потрапляє в загальний пул і може випасти або ні залежно від ліміту.

### Для типу `multiple` (кілька відповідей)

У колонці `Правильна_відповідь` вкажіть через кому: `Б,В` або `А,Г`

---

## 🎨 Рекомендація щодо кольорів у файлі

Для зручності редагування — фарбуйте кожну групу своїм кольором. Це не впливає на роботу системи, але значно спрощує перевірку файлу.

## 📁 Додавання тесту до проекту

1. Збережіть файл у папку `TESTS/`
2. Назва файлу: `предмет_клас_тема.xlsx` (латиницею, без пробілів)  
   Наприклад: `bio_6_klityna.xlsx`
3. Відкрийте `TESTS/manifest.json` і додайте рядок:

```json
{
  "file": "bio_6_klityna.xlsx",
  "id": "bio_6_klityna",
  "subject": "Біологія",
  "grade": 6,
  "title": "Будова клітини"
}
```

4. Збережіть `manifest.json`

## ⚠️ Часті помилки

| Помилка | Наслідок | Вирішення |
|---------|----------|-----------|
| Назва аркуша не `Питання` або `Мета` | Тест не завантажиться | Перейменуйте аркуш точно |
| Правильна відповідь `Б,В` з пробілом: `Б, В` | Може не зарахуватись | Без пробілів: `Б,В` |
| Порожній рядок між питаннями | Зупинка зчитування | Видаліть порожні рядки |
| Тип написаний з великої літери: `Single` | Не розпізнається | Пишіть малими: `single` |
